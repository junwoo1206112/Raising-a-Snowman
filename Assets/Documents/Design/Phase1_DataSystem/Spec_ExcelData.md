# Phase 1: Data System Specification

## 1. 개요
프로젝트 전반에서 사용할 밸런스 데이터를 엑셀에서 관리하고, 이를 유니티 `ScriptableObject`로 자동 변환하는 시스템을 구축한다.

## 2. 세부 사양
- **엑셀 양식**: 
  - `Stats`: 공격력, 체력, 공속, 방어력, 크리율, 크리 대미지 등 플레이어 능력치
  - `Items`: 무기 등급(6티어), 능력치 배율
  - `Monsters`: 일반 몬스터 10종 + 보스 3종 (체력, 골드 보상)
  - `Skills`: 액티브 스킬 5종 (대미지 배율, 쿨타임, 지속시간, 마나, 강화비용)
  - `Pets`: 펫 6종 (티어, 공격력, 공속, 버프타입, 해금 스테이지)
  - `Rebirth`: 환생 마일스톤 5단계 (필요 스테이지, 영구 공격력/골드 배율, 시작레벨 보너스)
  - `Quests`: 일일/주간 퀘스트 (조건 유형, 달성 수치, 보상 골드/아이템)
  - `Achievements`: 업적 8종 (조건 유형, 누적 수치, 보상 골드, 영구 스탯 보너스)
  - `Dungeons`: 특수 던전 (골드/아이템/펫/보스, 일일 입장 제한, 난이도 배율)
  - `OfflineRewards`: 오프라인 보상 기준값 (시간당 골드/경험치, 성장률)
- **데이터 컨버팅**:
  - `NPOI`를 사용하여 `.xlsx` 파일을 파싱.
  - 에디터 상단 메뉴 `Snowman > Data > Generate Template` 기능 제공.
  - `Snowman > Data > Import Excel` 기능 제공.
- **아키텍처 (DI)**:
  - `IDataService`: 데이터를 로드하고 제공하는 인터페이스.
  - `ExcelDataService`: 엑셀에서 데이터를 추출하여 SO로 변환 및 관리.
  - 직접적인 싱글톤 사용을 금지하며 모든 참조는 DI를 통해 주입받음.
